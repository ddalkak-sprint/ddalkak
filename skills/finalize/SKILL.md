# [5] 마무리 검증  (글랜·렉스)

파이프라인을 닫는 단계. 검증 결과를 사람이 읽기 좋게 정리하고 후속 조치를 정한다.

## 입력
- `.ddalkak/reports/<name>.verify.md`

## 동작
1. 검증에서 남은 불일치 항목 재확인.
2. 사용자에게 [수정 진행 / 그대로 확정 / 재검증] 확인.
3. `.ddalkak/ddalkak.config.json`에 완료 상태 기록.
4. 전체 파이프라인 최종 요약(무엇을 만들었고, 무엇이 남았는지) 보고.
