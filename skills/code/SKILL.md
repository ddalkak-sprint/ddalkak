# [3] code 추출  (퓨리)

`plan.md`를 실행 계획으로 삼아 파일을 생성/수정한다.

## 입력
- `.ddalkak/plan/<name>.plan.md`
- (선택) 프로젝트 루트 `design.md` — 팀 컨벤션

## 동작
1. plan.md의 파일 계획·구현 순서를 따른다.
2. design.md 컨벤션(스택/네이밍/구조)을 준수. 없으면 프로젝트 기존 패턴을 추론.
3. 브릿지 토큰을 코드 스타일로 매핑.
4. 생성 결과 요약을 사용자에게 보고.
